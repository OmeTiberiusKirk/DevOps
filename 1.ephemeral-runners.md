# Ephemeral runners

## Register and enter the Flag `--ephemeral`:
```
./config.sh --url https://github.com/YOUR-ORG/YOUR-REPO --token <YOUR_REGISTRATION_TOKEN> --ephemeral
```

## ขั้นตอนการสร้าง systemd Service
1. สร้างไฟล์ Service ขึ้นมาใหม่ในโฟลเดอร์ระบบด้วยคำสั่ง sudo: 
	```
	sudo vim /etc/systemd/system/github-runner.service
	```

2. คัดลอกคอนฟิกด้านล่างนี้ไปวาง (กรุณาปรับเปลี่ยนข้อมูลในส่วน User, WorkingDirectory และสิทธิ์ให้ตรงกับเครื่อง):
	```
	[Unit]
	Description=GitHub Actions Ephemeral Runner
	After=network.target

	[Service]
	# แนะนำให้รันด้วยสิทธิ์ User ธรรมดา (Hardening) ห้ามใช้ root เด็ดขาด
	User=ubuntu
	WorkingDirectory=/home/ubuntu/actions-runner

	# สั่งรันสคริปต์ของ GitHub Runner โดยตรง
	ExecStart=/home/ubuntu/actions-runner/run.sh

	# กลไกสำคัญ: บังคับให้ระบบเริ่มทำงานใหม่ทันทีหลังจากที่รัน Job เสร็จแล้วตัวสคริปต์ปิดตัวลง
	Restart=always
	RestartSec=2

	# ป้องกันสิทธิ์แบบเบื้องต้น (Hardening เผื่อมีโค้ดหลุดมาแก้ไข Service)
	ProtectSystem=full
	PrivateTmp=true

	[Install]
	WantedBy=multi-user.target
	```

3. ทำการ Reload daemon ของ systemd เพื่อให้ระบบรับทราบการสร้างไฟล์คอนฟิกใหม่:
	```
	sudo systemctl daemon-reload
	```

4. สั่งเปิดใช้งาน (Enable) เพื่อให้ Service ตัวนี้เปิดทำงานโดยอัตโนมัติทุกครั้งเมื่อมีการเปิดเครื่อง (Reboot):
	```
	sudo systemctl enable github-runner.service
	```
5. สั่งเริ่มทำงาน (Start) ระบบ Runner ทันที:
	```
	sudo systemctl start github-runner.service
	```
